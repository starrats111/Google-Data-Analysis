## 问题一：L7D佣金数据全为0

### 1.1 问题现象

wj02 账号的 L7D 分析弹窗中：
- L7D点击 ✅ 有数据（8, 40, 27...）
- L7D花费 ✅ 有数据（2.59, 2.90, 4.42...）
- 预算 ✅ 有数据
- 当前Max CPC ✅ 有数据
- **L7D佣金 ❌ 全部为 0.00**
- **L7D出单天数 ❌ 全部为 0**
- **保守EPC ❌ 全部为 0.00**
- **ROI ❌ 全部为 -1.00**

### 1.2 根因分析

**数据库类型：SQLite**

在 `api_analysis_service.py` 第782-783行：

```python
cast(AffiliateTransaction.transaction_time, Date) >= begin_date,
cast(AffiliateTransaction.transaction_time, Date) <= end_date,
```

SQLite 没有原生 DATE 类型，`CAST(datetime_value AS DATE)` 的行为是：

| 原始值 | CAST AS DATE | date() 函数 |
|--------|-------------|-------------|
| `2026-03-12 08:52:28.000000` | `2026` (仅年份!) | `2026-03-12` (正确) |

`"2026" >= "2026-03-06"` 字符串比较结果为 **FALSE**（因为 "2026" < "2026-"），导致**所有交易记录被过滤掉**。

### 1.3 验证数据

| 查询方式 | 结果 |
|----------|------|
| `cast(transaction_time, Date)` (当前代码) | **0 条** |
| `func.date(transaction_time)` (修复后) | **5311 条** |
| wj02 使用 func.date 查询 | **880 条交易，佣金 $2,665.93** |

### 1.4 修复方案

将第782-783行的 `cast(..., Date)` 替换为 `func.date(...)`：

```python
# 修复前（错误）
cast(AffiliateTransaction.transaction_time, Date) >= begin_date,
cast(AffiliateTransaction.transaction_time, Date) <= end_date,

# 修复后（正确）
func.date(AffiliateTransaction.transaction_time) >= begin_date.isoformat(),
func.date(AffiliateTransaction.transaction_time) <= end_date.isoformat(),
```

### 1.5 影响范围

- 仅影响 `backend/app/services/api_analysis_service.py` 一处
- 全局搜索确认 `cast(..., Date)` 仅此一处使用
- 修复后所有用户的 L7D 佣金数据将恢复正常

## 问题三：广告创建 AI 智能文案不响应（401错误）

### 3.1 问题现象

广告创建向导 → Step 2「AI 智能文案」步骤中：
- 页面显示「正在分析历史数据...」和「等待 AI 响应...」
- 界面一直卡在加载状态，AI 始终无响应
- **DevTools 控制台报错：`Failed to load resource: the server responded with a status of 401`**

### 3.2 根因分析

**问题出在前端 Token 获取方式不一致。**

本系统的认证架构采用了安全的内存Token方案（SEC-7）：
- 登录时，`authStore.js` 调用 `setHolderToken(access_token)` 将 Token 存入 **内存**（`tokenHolder.js`）
- `localStorage` 中只存储了 `user`（用户信息），**从未存储过 `token`**
- 普通 axios 请求通过 `api.js` 拦截器自动从 `getToken()` 获取内存中的 Token

**但是** `AdCreationWizard.jsx` 中的 SSE 流式请求绕过了 axios，直接使用原生 `fetch`，并错误地从 `localStorage` 获取 Token：

```169:177:frontend/src/pages/ads/AdCreationWizard.jsx
    const token = localStorage.getItem('token')   // ← 这里永远返回 null
    const baseUrl = api.defaults.baseURL || ''

    try {
      const resp = await fetch(`${baseUrl}/api/ad-creation/generate-ad-copy-stream`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${token}`,          // ← 变成 "Bearer null"
```

**对比正确的写法**（同项目 `articleApi.js` 中的 SSE 请求）：

```39:40:frontend/src/services/articleApi.js
    const token = getToken()                         // ← 正确：从内存获取
    if (token) headers['Authorization'] = `Bearer ${token}`
```

### 3.3 完整调用链路分析

| 环节 | 实际情况 | 预期情况 |
|------|----------|----------|
| 1. 用户登录 | `authStore` 调用 `setHolderToken(token)` 存入内存 ✅ | — |
| 2. 普通API请求 | `api.js` 拦截器调用 `getToken()` 从内存取 ✅ | — |
| 3. AI文案SSE请求 | `localStorage.getItem('token')` → **返回 null** ❌ | 应调用 `getToken()` |
| 4. 请求头 | `Authorization: Bearer null` ❌ | `Authorization: Bearer <有效JWT>` |
| 5. 后端验证 | JWT解码 `null` 失败 → **返回 401** | 解码成功 → 返回用户信息 |

### 3.4 修复方案

修改 `frontend/src/pages/ads/AdCreationWizard.jsx`：

**第一步**：在文件顶部添加 import：
```javascript
import { getToken } from '../../services/tokenHolder'
```

**第二步**：将第 169 行：
```javascript
const token = localStorage.getItem('token')
```
改为：
```javascript
const token = getToken()
```

### 3.5 影响范围

- 仅影响 `frontend/src/pages/ads/AdCreationWizard.jsx` 一处
- 同项目其他 SSE 请求（如 `articleApi.js`）已正确使用 `getToken()`
- 修复后广告创建的 AI 智能文案功能将正常工作
