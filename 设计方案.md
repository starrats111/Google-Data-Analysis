> **状态：** ✅ 代码已完成，⏳ 待部署验收

#### 问题诊断

通过 SSH 到服务器查询数据库，对比 07 提供的截图数据，精确定位问题：

- **数据来源：** yz01 用户，MCC 货币为 CNY

## CR-043：文章发布反爬虫修复

### 问题描述

文章发布时通过 `merchant_crawler.py` 爬取商家网站，经常触发反爬虫检测（Cloudflare 5s盾、403拒绝等）。

### 根因分析（8个问题点）

| # | 问题 | 文件 | 严重度 |
|---|------|------|--------|
| 1 | HEADERS User-Agent 过旧 (Chrome/124, 2024年) | merchant_crawler.py:107-125 | P0 |
| 2 | Accept-Encoding 缺少 Brotli (br) | merchant_crawler.py:115 | P0 |
| 3 | curl_cffi impersonate 版本过旧 (chrome110/edge101) | merchant_crawler.py:689 | P0 |
| 4 | _build_stealth_headers Sec-Fetch-Site 设为 cross-site，应为 none | merchant_crawler.py:609 | P1 |
| 5 | 子页面爬取间隔过短 (0.5-1.5秒)，最多爬8页 | merchant_crawler.py:969-976 | P1 |
| 6 | 每次请求创建新 httpx.Client，Cookie 不连续 | merchant_crawler.py:818 | P1 |
| 7 | 图片下载 UA 过旧 (Chrome/120)，Referer 设为图片自身URL | remote_publisher.py:322-326 | P1 |
| 8 | Playwright 只检查 python3，不检查 python | merchant_crawler.py:766 | P2 |

### 修复内容

| 修改 | 文件 | 描述 |
|------|------|------|
| Fix-1 | merchant_crawler.py | HEADERS UA → Chrome/135, Sec-Ch-Ua → v135, Accept-Encoding 加 br |
| Fix-2 | merchant_crawler.py | _FALLBACK_UAS 全部更新到 Chrome 134/135, Firefox 136, Edge 135 |
| Fix-3 | merchant_crawler.py | curl_cffi targets → chrome131/133, safari18_0, edge131 |
| Fix-4 | merchant_crawler.py | _build_stealth_headers: 加 br, Sec-Fetch-Site → none, 默认版本 → 135 |
| Fix-5 | merchant_crawler.py | 子页面间隔 → 2.0-4.0秒，最大数量 → 5 |
| Fix-6 | merchant_crawler.py | Playwright 先查 python3 再查 python |
| Fix-7 | remote_publisher.py | 图片下载 UA → Chrome/135, Referer → 域名根路径, 加 Sec-Fetch-* |

## CR-044：商家管理权限开放给员工

### 问题描述

员工(member角色)进入商家管理页面只能看到"我的商家" tab，无法看到商家目录，导致无法执行核心业务流程：**查看商家 → 领取商家 → 创建广告**。

### 根因

前端 `canManage = role === 'manager' || role === 'leader'` 控制了统计卡片、商家目录、违规管理、推荐管理等所有 tab 的显示。

### 修复内容

**原则：员工可"看"所有数据并"领取"商家，"写"操作仍限经理/组长**

| 修改 | 文件 | 描述 |
|------|------|------|
| Fix-1 | MerchantManagement.jsx | 移除统计卡片的 canManage 守卫，所有用户可见 |
| Fix-2 | MerchantManagement.jsx | 移除商家目录/待补MID/违规/推荐 tab 的 canManage 守卫 |
| Fix-3 | MerchantManagement.jsx | useEffect 初始化：所有角色都加载 stats + merchants |
| Fix-4 | MerchantManagement.jsx | activeKey 不再强制员工只看 assignments |
| Fix-5 | merchants.py | ranking API 从 get_current_manager_or_leader → get_current_user |
| Fix-6 | MerchantPerformance.jsx | 移除 isEmployee 对 ranking 的跳过 |

**保持不变的权限：**
- 编辑商家/分配给员工：canManage (manager/leader)
- 平台同步/转移分配/修复MID：isManager (manager)
- 领取商家：get_current_user（已全角色开放）
