## 一、问题诊断

### 1.1 核心问题：L7D 数据不完整

**现象：** 标黄的L7D数据（佣金、订单数）与数据中心实际值不符

**根本原因：** `api_only_analysis_service.py` 第 200-210 行的数据聚合逻辑过于严格

```python
# ❌ 问题代码
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        if pd.date in campaign_data["dates"]:  # ⚠️ 只统计有Google Ads数据的日期
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
```

**问题分析：**

| 场景 | Google Ads 数据 | 平台数据 | 当前逻辑 | 正确逻辑 |
|------|-----------------|---------|---------|---------|
| 正常日期 | ✅ 有点击/花费 | ✅ 有订单/佣金 | ✅ 统计 | ✅ 统计 |
| 平台延迟 | ✅ 有点击/花费 | ❌ 无订单/佣金 | ✅ 统计 0 | ✅ 统计 0 |
| **数据缺失** | ✅ 有点击/花费 | ✅ 有订单/佣金 | ❌ **漏统计** | ✅ 统计 |
| 平台独立 | ❌ 无点击/花费 | ✅ 有订单/佣金 | ❌ **漏统计** | ✅ 统计 |

**数据缺失原因：**
- Google Ads 数据和平台数据来自不同系统，同步时间不同步
- 某些日期可能只有平台数据，没有Google Ads数据（或反之）
- 当前逻辑要求"既有Google Ads数据，又有平台数据"才统计，导致单边数据被丢弃

### 1.2 数据来源错误

**问题：** 平台代码匹配不准确

| 问题 | 影响 |
|------|------|
| `extracted_platform_code` 大小写不一致 | 无法找到对应 `AffiliatePlatform` |
| 商家ID（MID）不匹配 | 无法关联到正确的 `AffiliateAccount` |
| 平台代码标准化不完整 | 某些平台的代码变体无法识别 |

## 二、修复方案

### 2.1 修复数据聚合逻辑（优先级：🔴 高）

**文件：** `backend/app/services/api_only_analysis_service.py`

**修改位置：** 第 200-210 行

**修改前：**
```python
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        if pd.date in campaign_data["dates"]:  # ❌ 过于严格
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
            if pd.orders and pd.orders > 0:
                order_days.add(pd.date)
```

**修改后：**
```python
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        # ✅ 使用日期范围而非集合匹配，确保完整统计
        if begin_date <= pd.date <= end_date:
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
            if pd.orders and pd.orders > 0:
                order_days.add(pd.date)
```

**改动说明：**
- 从 `if pd.date in campaign_data["dates"]` 改为 `if begin_date <= pd.date <= end_date`
- `campaign_data["dates"]` 只包含有Google Ads数据的日期
- 改为日期范围后，会统计L7D内所有平台数据，即使没有对应的Google Ads数据

**影响范围：**
- ✅ L7D 佣金数据完整性提升
- ✅ L7D 订单数数据完整性提升
- ✅ 出单天数计算准确
- ⚠️ 可能导致某些日期的平台数据被重复统计（需要验证数据质量）

### 2.2 修复平台代码匹配（优先级：🟡 中）

**文件：** `backend/app/services/api_only_analysis_service.py`

**修改位置：** `_find_affiliate_account()` 方法

**问题代码：**
```python
# 按平台代码查找（platform_code = "rw"/"pm"/"lh" 等）
query = self.db.query(AffiliateAccount).join(
    AffiliatePlatform
).filter(
    AffiliateAccount.user_id == user_id,
```

**修复方案：**

1. **标准化平台代码**
```python
def _normalize_platform_code(code: str) -> str:
    """标准化平台代码，处理大小写和变体"""
    if not code:
        return None
    
    code = code.upper()
    
    # 平台代码映射表（处理变体）
    PLATFORM_ALIASES = {
        'CF': 'CF',      # CreatorFlare
        'CG': 'CG',      # CollabGlow
        'BSH': 'BSH',    # BrandSparkHub
        'PM': 'PM',      # PartnerMatic
        'LB': 'LB',      # LinkBux
        'LH': 'LH',      # LinkHaiTao
        'RW': 'RW',      # Rewardoo
        'CREATORFLARE': 'CF',
        'COLLABGLOW': 'CG',
        'BRANDSPARHUB': 'BSH',
        'PARTNERMATIC': 'PM',
        'LINKBUX': 'LB',
        'LINKHAITAO': 'LH',
        'REWARDOO': 'RW',
    }
    
    return PLATFORM_ALIASES.get(code, code.lower())
```

2. **改进 `_find_affiliate_account()` 方法**
```python
def _find_affiliate_account(
    self,
    platform_code: Optional[str],
    merchant_id: Optional[str],
    user_id: int,
    account_id: Optional[int] = None
) -> Optional[AffiliateAccount]:
    """查找对应的联盟账号 - 使用标准化平台代码"""
    if not platform_code:
        return None
    
    # ✅ 标准化平台代码
    normalized_code = self._normalize_platform_code(platform_code)
    
    # 如果指定了账号ID，直接查找
    if account_id:
        account = self.db.query(AffiliateAccount).filter(
            AffiliateAccount.id == account_id,
            AffiliateAccount.user_id == user_id,
            AffiliateAccount.is_active == True
        ).first()
        if account:
            # ✅ 验证平台代码是否匹配（使用标准化后的代码）
            if account.platform and account.platform.platform_code.upper() == normalized_code:
                return account
        return None
    
    # ✅ 按标准化平台代码查找
    query = self.db.query(AffiliateAccount).join(
        AffiliatePlatform
    ).filter(
        AffiliateAccount.user_id == user_id,
        AffiliatePlatform.platform_code == normalized_code,
        AffiliateAccount.is_active == True
    )
    
    # ✅ 如果提供了商家ID，优先按商家ID匹配
    if merchant_id:
        query = query.filter(AffiliateAccount.account_code == merchant_id)
        account = query.first()
        if account:
            return account
    
    # 如果没找到匹配的账号代码，返回该平台下的第一个账号
    return query.first()
```

**改动说明：**
- 新增 `_normalize_platform_code()` 方法处理平台代码变体
- 改进平台代码匹配逻辑，支持大小写不敏感
- 优先按商家ID匹配，提高准确性

```python
检查：
1. Google Ads 数据和平台数据的日期覆盖
2. 平台代码匹配情况
3. 商家ID匹配情况
4. 数据缺失情况

sys.path.insert(0, '.')

from datetime import date, timedelta

from app.models.user import User
from app.models.affiliate_account import AffiliateAccount, AffiliatePlatform
from app.models.platform_data import PlatformData

    print("❌ 用户 wj02 不存在")
    sys.exit(1)

print(f"✅ 用户: {user.username} (ID: {user.id})")

# L7D 日期范围
end_date = date.today() - timedelta(days=1)
begin_date = end_date - timedelta(days=6)

google_dates = set(gd.date for gd in google_data)
print(f"  覆盖日期数: {len(google_dates)}")
print(f"  日期范围: {min(google_dates) if google_dates else 'N/A'} ~ {max(google_dates) if google_dates else 'N/A'}")

# 2. 检查平台数据
print(f"\n💰 平台数据统计:")
platform_data = db.query(PlatformData).filter(
    PlatformData.user_id == user.id,
    PlatformData.date >= begin_date,
    PlatformData.date <= end_date
).all()

print(f"  总记录数: {len(platform_data)}")
platform_dates = set(pd.date for pd in platform_data)
print(f"  覆盖日期数: {len(platform_dates)}")
print(f"  日期范围: {min(platform_dates) if platform_dates else 'N/A'} ~ {max(platform_dates) if platform_dates else 'N/A'}")

# 3. 检查日期覆盖差异
print(f"\n🔍 日期覆盖分析:")
only_google = google_dates - platform_dates
only_platform = platform_dates - google_dates
both = google_dates & platform_dates

print(f"  仅有 Google Ads 数据的日期: {len(only_google)} 天 → {sorted(only_google)}")
print(f"  仅有平台数据的日期: {len(only_platform)} 天 → {sorted(only_platform)}")
print(f"  两者都有的日期: {len(both)} 天")

# 4. 检查平台代码匹配
print(f"\n🏷️  平台代码匹配分析:")
platform_codes = set()
for gd in google_data:
    if gd.extracted_platform_code:
        platform_codes.add(gd.extracted_platform_code)

for code in platform_codes:
    platform = db.query(AffiliatePlatform).filter(
        AffiliatePlatform.platform_code == code.lower()

                AffiliatePlatform.id == acc.platform_id

            print(f"      - 平台: {platform.platform_code if platform else 'N/A'}, 账号: {acc.account_name}")
    else:

print("\n✅ 诊断完成")
```

## 六、修复详解

### 6.1 修复前后对比
