# 平台API配置 - 最简单教程（不报错、不乱码）

## 🎯 目标
让任何人都能成功配置平台API，不会报错，不会乱码。

---

## 📝 第一步：找到你的账号

1. 登录系统
2. 点击左侧菜单 **"平台账号"**
3. 找到你要配置的账号（比如 "wenjun"）
4. 点击账号右侧的 **"编辑"** 按钮

---

## 🔧 第二步：配置API（三种方法，选最简单的一种）

### ⭐ 方法1：只填API Token（最简单，推荐）

**适用场景**：大多数情况，平台使用默认API地址

**操作步骤**：
1. 在编辑账号页面，找到 **"API Token"** 输入框
2. 直接粘贴你的API Token（一串字母数字）
3. 点击 **"确定"** 保存

**示例**：
```
API Token: 41df7906743fceacaa16ca25abfa1ce4
```

**就这么简单！** ✅

---

### 方法2：在备注中配置（如果方法1不行）

**适用场景**：平台有多个渠道，需要指定不同的API地址

**操作步骤**：

#### 步骤1：打开备注框
在编辑账号页面，找到 **"备注"** 输入框（多行文本框）

#### 步骤2：复制下面的模板（根据你的平台选择）

**如果是 RW 平台（Rewardoo）**：
```json
{"api_token": "你的token", "rewardoo_api_url": "https://api.rewardoo.com/api"}
```

**如果是 CG 平台（CollabGlow）**：
```json
{"api_token": "你的token", "collabglow_api_url": "https://api.collabglow.com/api"}
```

**如果是 LH 平台（LinkHaitao）**：
```json
{"api_token": "你的token"}
```

#### 步骤3：替换内容
- 把 `你的token` 替换成你的真实API Token
- 把API地址替换成平台给你的地址（如果没有特殊要求，就用上面的默认地址）

#### 步骤4：保存
点击 **"确定"** 按钮

**完整示例（RW平台）**：
```json
{"api_token": "41df7906743fceacaa16ca25abfa1ce4", "rewardoo_api_url": "https://api.rewardoo.com/api"}
```

---

### 方法3：同步时临时输入（测试用）

**适用场景**：想先测试一下，不保存到账号

**操作步骤**：
1. 点击账号右侧的 **"同步数据"** 按钮
2. 在弹出的对话框中：
   - 输入 **开始日期**（格式：2026-01-01）
   - 输入 **结束日期**（格式：2026-01-31）
   - 输入 **API Token**（可选，如果不填就用账号里保存的）
3. 点击 **"确定"** 开始同步

---

## ✅ 第三步：测试配置

### 测试方法1：直接同步数据
1. 点击账号右侧的 **"同步数据"** 按钮
2. 选择日期范围（建议选最近7天）
3. 点击 **"确定"**
4. 如果看到 **"同步成功"**，说明配置正确 ✅
5. 如果看到错误提示，继续看下面的"常见错误解决"

### 测试方法2：查看错误信息
如果同步失败，系统会显示错误信息：
- **404错误**：API地址不对，检查备注中的URL
- **401错误**：Token不对，检查Token是否正确
- **500错误**：服务器问题，稍后重试

---

## 🚨 常见错误解决

### 错误1：API端点不存在(404)

**原因**：API地址配置错误

**解决方法**：
1. 检查备注中的API地址是否正确
2. 联系平台客服，确认正确的API地址
3. 更新备注中的URL

**示例（RW平台，如果默认地址不行）**：
```json
{"api_token": "你的token", "rewardoo_api_url": "https://api-channel1.rewardoo.com/api"}
```

---

### 错误2：未授权(401)

**原因**：API Token错误或过期

**解决方法**：
1. 登录平台后台，重新生成Token
2. 复制新的Token
3. 更新到账号的API Token字段或备注中

---

### 错误3：同步失败(500)

**原因**：服务器问题或网络问题

**解决方法**：
1. 等待几分钟后重试
2. 检查网络连接
3. 如果持续失败，联系技术支持

---

## 📋 配置检查清单

配置完成后，检查以下项目：

- [ ] API Token已填写（在API Token字段或备注中）
- [ ] 备注格式正确（如果是JSON格式，确保没有语法错误）
- [ ] 日期格式正确（YYYY-MM-DD，例如：2026-01-31）
- [ ] 点击"同步数据"后，能看到同步结果

---

## 💡 重要提示

### ✅ 推荐做法
1. **优先使用方法1**（只填API Token），最简单
2. **如果方法1不行，再用方法2**（在备注中配置）
3. **配置后立即测试**，确保能同步数据

### ❌ 避免的错误
1. **不要**在备注中写中文说明（只写JSON）
2. **不要**在JSON中加注释（`//` 或 `/* */`）
3. **不要**用单引号（`'`），必须用双引号（`"`）
4. **不要**在JSON最后加逗号（`,`）

### ✅ 正确的备注格式
```json
{"api_token": "abc123", "rewardoo_api_url": "https://api.rewardoo.com/api"}
```

### ❌ 错误的备注格式
```json
// 这是注释，不能写
{"api_token": 'abc123',  // 不能用单引号，不能写注释
 "rewardoo_api_url": "https://api.rewardoo.com/api",  // 最后不能有逗号
}
```
