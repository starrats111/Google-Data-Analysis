## 一、问题描述

### 现象
标黄的 L7D 数据（佣金、订单数、出单天数）与数据中心实际值不符，导致数据分析结果不准确。

### 根本原因
`api_only_analysis_service.py` 中的数据聚合逻辑过于严格，只统计既有 Google Ads 数据又有平台数据的日期，导致单边数据被丢弃。

**具体问题代码（第 200-210 行）：**
```python
# ❌ 问题代码
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        if pd.date in campaign_data["dates"]:  # ⚠️ 只统计有Google Ads数据的日期
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
```

**问题分析：**
- `campaign_data["dates"]` 只包含有 Google Ads 数据的日期
- 如果某个日期只有平台数据，没有 Google Ads 数据，则该日期的平台数据被漏统计
- 导致 L7D 佣金、订单数等数据不完整

## 二、修复方案

### 2.1 修复数据聚合逻辑（优先级：🔴 高）

**文件：** `backend/app/services/api_only_analysis_service.py`  
**修改位置：** 第 200-210 行

**修改前：**
```python
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        if pd.date in campaign_data["dates"]:  # ❌ 过于严格
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
```

**修改后：**
```python
for pd in platform_data_list:
    if pd.affiliate_account_id == affiliate_account.id:
        # ✅ 使用日期范围而非集合匹配，确保完整统计
        if begin_date <= pd.date <= end_date:
            total_commission += pd.commission or 0
            total_orders += pd.orders or 0
            if pd.orders and pd.orders > 0:
                order_days.add(pd.date)
```

**改动说明：**
- 从 `if pd.date in campaign_data["dates"]` 改为 `if begin_date <= pd.date <= end_date`
- 改为日期范围后，会统计 L7D 内所有平台数据，即使没有对应的 Google Ads 数据
- 确保 L7D 佣金、订单数、出单天数等数据完整准确

### 2.2 修复平台代码匹配（优先级：🟡 中）

**文件：** `backend/app/services/api_only_analysis_service.py`  
**修改位置：** `_find_affiliate_account()` 方法（第 240-260 行）

**改进点：**
- 使用 `func.lower()` 进行数据库级别的大小写不敏感比较
- 标准化输入的平台代码（`.lower().strip()`）
- 优先按商家ID匹配，提高准确性

**修改后的代码：**
```python
def _find_affiliate_account(
    self,
    platform_code: Optional[str],
    merchant_id: Optional[str],
    user_id: int,
    account_id: Optional[int] = None
) -> Optional[AffiliateAccount]:
    """查找对应的联盟账号 - 使用标准化平台代码匹配（大小写不敏感）"""
    if not platform_code:
        return None
    
    # ✅ 标准化平台代码（大小写不敏感）
    normalized_code = platform_code.lower().strip()
    
    # 如果指定了账号ID，直接查找
    if account_id:
        account = self.db.query(AffiliateAccount).filter(
            AffiliateAccount.id == account_id,
            AffiliateAccount.user_id == user_id,
            AffiliateAccount.is_active == True
        ).first()
        if account:
            # ✅ 验证平台代码是否匹配（使用标准化后的代码）
            if account.platform and account.platform.platform_code.lower() == normalized_code:
                return account
        return None
    
    # ✅ 按标准化平台代码查找（platform_code = "rw"/"pm"/"lh" 等）
    query = self.db.query(AffiliateAccount).join(
        AffiliatePlatform
    ).filter(
        AffiliateAccount.user_id == user_id,
        # ✅ 使用 func.lower() 进行数据库级别的大小写不敏感比较
        func.lower(AffiliatePlatform.platform_code) == normalized_code,
        AffiliateAccount.is_active == True
    )
    
    # ✅ 如果提供了商家ID，优先按商家ID匹配
    if merchant_id:
        query = query.filter(AffiliateAccount.account_code == merchant_id)
        account = query.first()
        if account:
            return account
    
    # 如果没找到匹配的账号代码，返回该平台下的第一个账号
    base_query = self.db.query(AffiliateAccount).join(
        AffiliatePlatform
    ).filter(
        AffiliateAccount.user_id == user_id,
        func.lower(AffiliatePlatform.platform_code) == normalized_code,
        AffiliateAccount.is_active == True
    )
    return base_query.first()
```
