Advertiser Basic API
API Documentation  Advertiser Basic API
Summary
You can use this API to check advertiser details and acquire tracking links to monetize your traffic.
Update History
Update Date	Update Information
2023/08/24	The Advertiser Basic API 2.0 has added a request parameter 'merchant_status', and it has expanded the data format type for the 'type' input to include 'archive'. The returned data is advertiser information whose cooperation relationship with the publisher is approved.
Request Information
URL	https://www.linkhaitao.com/api.php?mod=medium&op=merchantBasicList3
Return Format	json
Request Method	GET/POST
Request Parameters
Required Parameter	Description
token

Optional Parameter	Description
type	Return Data Format Types: JSON, Archive (Returns a compressed file in CSV format) {Optional, the default is json} When 'type' is set to 'archive', the response will contain the entire dataset without pagination. At least two requests are required: the first request creates a task and returns a task ID, while the second request uses the task ID to request the compressed archive file. (While the archive is still being generated, the status will be 'pending', and the file path will be empty. Once the generation is complete, the status will change to 'done', and the file path will be a valid URL.) First request: return {'task_id': '10001'} The second request: task_id=10001 {'status': 'pending/done','file_url': ''}
merchant_status	Advertiser status: 0 | offline, 1 | online (The default is online)
is_auto	Auto-approved advertiser or not: 0 No, 1 Yes (Return all if it is empty)
per_page	Pagesize number of pages (Minimum 100, maximum 40000)
page	Page (Default is 1)

datetime	Date joined/removed from the advertiser
relationship	Your advertiser relationship
tracking_url	Your advertiser tracking link
tracking_url_short	Your advertiser short link
RD	Return day
site_desc	Website description
restricted_keywords	Restricted keywords
currency	Currency
support_region	Support region
support_deeplink	Whether support deep link
approval_mode	Approval mode
traffic_promotion_type	Only allowed; Only restricted
promotion_area	When 'traffic_promotion_type' is 'only allowed', the 'promotion_area' is the areas that can be promoted; When 'traffic_promotion_type' is 'only restricted', the 'promotion_area' is the areas that cannot be promoted.
Result Code

0	Success
4001	Date format is incorrect
4002	The start date cannot be less than the end date
9999	Request frequency limit
