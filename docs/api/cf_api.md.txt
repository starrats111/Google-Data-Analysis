Transaction API
Summary
You can use this API to pull detailed transactions, commission status and many more.

Updates
Name
Description
v1.0
2025/12/01— Add Transaction API
Request Information
Name
Description
Website
https://api.creatorflare.com/api/transaction
Return Format
JSON
HTTP Request
POST application/json
Required Parameters
Name
Description
source
creatorflare
token
beginDate
Transaction period. Format: YYYY-MM-DD(Data queries are only supported for dates starting from 2023/1/1.). Either transaction period or validation period is mandatory.
endDate
Format: YYYY-MM-DD
beginApproveDate
Validation period. Format: YYYY-MM-DD(Data queries are only supported for dates starting from 2023/1/1.). Either transaction period or validation period is mandatory.
endApproveDate
Format: YYYY-MM-DD
Optional Parameters to Filter
Name
Description
dataScope
Data Retrieval Range. The default value is "channel", which returns data only for the single channel associated with the current token. If set to "user", it retrieves data for all channels belonging to the user.
orderId
Order ID
status
Commission Status,The value for this parameter must be of the array data type.Type: List Eg: ["All"]
uid
uid, your custom tracking variable.
brandId
Brand ID, a series of numbers, like 66303
mcid
Unique identifier for the brand, a series of characters, like ulike0
curPage
Current page index
perPage
Number of transactions shown per page - max 2000 transactions per page
Return parameters
Name
Description
code
Response status code (0 = success)
message
Response status description (e.g., "success")
data
Main data container
├─ total
Total number of records
├─ curPage
Current page number (starting from 1)
├─ totalPage
Total number of pages
├─ hasNext
Whether next page exists (true/false)
└─ list
Array of transactions details
├─ ​​creatorflare_id
Creatorflare system unique ID
├─ ​​brand_id
Brand ID, a series of numbers, like 66303
├─ ​​mcid
Unique identifier for the brand, a series of characters, like ulike0
├─ ​​merchant_name
Brand Name
├─ ​​order_id
Order ID
├─ ​​order_time
Transaction Time
├─ ​​sale_amount
Sale Amount
├─ ​​sale_comm
Sale Commission
├─ ​​status
Commission Status
├─ ​​uid
uid, your custom tracking variable
├─ ​​uid2
uid2, your custom tracking variable
├─ ​​uid3
uid3, your custom tracking variable
├─ ​​uid4
uid4, your custom tracking variable
├─ ​​uid5
uid5, your custom tracking variable
├─ ​​click_ref
Unique Click Id
├─ ​​prod_id
Product ID
├─ ​​order_unit
Order Unit Amount
├─ ​​comm_rate
Commission Rate
├─ ​​validation_date
Date the transaction validated
├─ ​​note
Reason for status change
├─ ​​customer_country
Customer country. If empty, it means the merchant does not provide
├─ ​​voucher_code
Code used at checkout
├─ ​​paid_status
Whether this transaction has been paid to publishers. 1 = paid. 0 = not paid.
├─ ​​last_update_time
The latest time we update the transaction status from the advertiser.
├─ settlement_date
Date when commission is approved and ready for you to withdraw
├─ paid_date
Date when the payment is marked paid. Format: YYYY-MM-DD
Result Code
Return Code
Description
0
Success
1000
Publisher does not exist
1001
Invalid token
1002
Call frequency too high
1006
Query time span cannot exceed 62 days
10001
Missing required parameters or incorrect format
Request Format
curl --location --request POST 'https://api.creatorflare.com/api/transaction' \
--header 'Content-Type: application/json' \
--data-raw '
  {
    "source": "creatorflare",

    "dataScope": "user",
    "beginDate": "2025-06-01",
    "endDate": "2025-06-05",
    "curPage": 1,
    "perPage": 20
  }
'
