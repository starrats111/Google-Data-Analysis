# 员工配置MCC账号 - 超简单指南

## 📋 前提条件

- 你已经有了Google Ads MCC账号ID（例如：`941-949-6301`）
- 经理已经配置了共享的Google Ads API配置（客户端ID、密钥、开发者令牌）

---

## 🚀 三步完成配置

### 第1步：获取刷新令牌（Refresh Token）

**🎉 推荐方式：使用在线工具（最简单！）**

👉 **查看详细教程：`docs/员工获取刷新令牌_在线工具版.md`**

**超简单3步：**
1. 在浏览器中访问：`https://google-data-analysis.top/oauth-tool`
2. 填写客户端ID和密钥（如果系统已自动填充，直接点击"生成授权URL"）
3. 完成Google授权，复制刷新令牌

**✅ 优势：**
- 不需要安装Python
- 不需要访问本地文件
- 不需要命令行操作
- 只需要浏览器

---

**备选方式：本地运行脚本**

**Windows用户：**
👉 **查看超详细教程：`docs/员工获取刷新令牌_Windows傻瓜教程.md`**
👉 **或快速复制版：`docs/员工获取刷新令牌_快速复制版.md`**

**Linux/Mac用户：**
```bash
export GOOGLE_ADS_SHARED_CLIENT_ID="经理提供的客户端ID"
export GOOGLE_ADS_SHARED_CLIENT_SECRET="经理提供的客户端密钥"
python backend/scripts/get_google_ads_refresh_token.py
```

---

### 第2步：在系统中添加MCC账号

1. **登录系统**
2. **进入"MCC账号"页面**（左侧菜单）
3. **点击"添加MCC账号"按钮**
4. **填写信息**：
   - ✅ **MCC账号ID**：你的Google Ads MCC账号ID（必填）
   - ✅ **刷新令牌**：从第1步获取的刷新令牌（必填）
   - ⭕ **账号名称**：可选，给你的账号起个名字
   - ⭕ **邮箱**：可选
   - ⭕ **客户端ID**：如果系统显示需要填写，问经理要
   - ⭕ **客户端密钥**：如果系统显示需要填写，问经理要
   - ⭕ **开发者令牌**：如果系统显示需要填写，问经理要
5. **点击"确定"保存**

---

### 第3步：测试连接

1. 在MCC账号列表中，找到你刚添加的账号
2. 点击"测试连接"按钮
3. 如果显示"连接测试成功"，说明配置正确 ✅
4. 如果显示错误，检查：
   - MCC账号ID是否正确
   - 刷新令牌是否正确复制（没有多余空格）
   - 是否使用了正确的Google账号授权

---

## ❓ 常见问题

### Q1: 系统提示"客户端ID不能为空"怎么办？

**A**: 说明经理还没有配置共享配置。你需要：
- 向经理要客户端ID、客户端密钥、开发者令牌
- 在添加MCC账号时填写这些信息

### Q2: 我没有Google Cloud账号怎么办？

**A**: 如果经理配置了共享配置，你**不需要**Google Cloud账号，只需要：
- 获取刷新令牌（使用经理提供的客户端ID和密钥）
- 在系统中添加MCC账号

### Q3: 刷新令牌过期了怎么办？

**A**: 重新运行脚本获取新的刷新令牌，然后在系统中：
- 点击"编辑"按钮
- 更新刷新令牌
- 保存

### Q4: 测试连接失败怎么办？

**A**: 检查以下几点：
1. MCC账号ID是否正确（格式：`123-456-7890`）
2. 刷新令牌是否正确（完整复制，没有多余空格）
3. 你的Google账号是否有权限访问这个MCC账号
4. 联系经理确认共享配置是否正确

---

## 📝 经理需要做什么

为了让员工使用最简单的配置方式，经理需要：

1. **在后端配置文件中设置共享配置**（`.env`文件或环境变量）：
   ```
   GOOGLE_ADS_SHARED_CLIENT_ID=你的客户端ID
   GOOGLE_ADS_SHARED_CLIENT_SECRET=你的客户端密钥
   GOOGLE_ADS_SHARED_DEVELOPER_TOKEN=你的开发者令牌
   ```

2. **告诉员工**：
   - 客户端ID
   - 客户端密钥
   - （可选）开发者令牌（如果配置了环境变量就不需要）

3. **重启后端服务**，让配置生效
