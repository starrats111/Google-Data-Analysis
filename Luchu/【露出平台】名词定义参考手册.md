## 一、核心业务术语

### 露出（Exposure）

**定义**: 通过在博客网站发布推广性质的文章，植入商家追踪链接和品牌信息，以达到引流和转化目的的营销行为。

**特点**:
- 内容形式为博客文章
- 包含商家追踪链接
- 自然融入品牌关键词
- 符合SEO优化要求
- 对读者有实际价值

**示例**: 在VitaHaven网站发布一篇"2026年最佳护肤品推荐"文章，其中自然推荐某品牌产品并植入追踪链接。

---

### 露出指南（Exposure Guidelines）

**定义**: 团队内部制定的露出内容创作规范，确保内容质量和合规性。

**核心原则**:
| 原则 | 说明 |
|------|------|
| 准确性 | 信息真实可靠 |
| 时效性 | 内容与时俱进 |
| 人性化 | 语言自然，像真人写的 |
| 软植入 | 推广内容自然融入 |
| 可搜索 | 符合SEO优化 |

**文件位置**: `D:\Luchu\露出指南.txt`

---

### 露出提示词（Exposure Prompt）

**定义**: 用于指导AI（Claude）生成露出内容的标准化提示词模板。

**包含内容**:
- 商家网站分析要求
- 链接植入规则
- 核心写作原则
- 文章结构要求
- 图片编号规范
- 产品模块格式
- 输出格式规范

**文件位置**: `D:\Luchu\露出提示词.txt`

---

### 商家URL（Merchant URL）

**定义**: 需要推广的商家网站或产品页面的原始链接。

**用途**:
- AI分析商家信息
- 提取产品图片
- 获取产品描述
- 了解品牌定位

**示例**: `https://www.example-brand.com/products/skincare-set`

---

### 追踪链接（Tracking Link / Affiliate Link）

**定义**: 包含追踪参数的推广链接，用于记录流量来源和转化数据。

**特点**:
- 包含唯一标识符
- 可追踪点击和转化
- 与佣金结算相关

**示例**: `https://track.example.com/aff?id=wj07&campaign=skincare2026`

---

### 品牌关键词（Brand Keyword）

**定义**: 需要在文章中自然出现的商家品牌名称或核心产品词。

**默认设置**: 每篇文章出现 **10次**（可调整）

**植入原则**:
- 分布均匀，不集中堆砌
- 语境自然，不生硬
- 变换形式（品牌名/产品名/简称等）

## 二、内容相关术语

### 博客文章（Blog Article / Post）

**定义**: 发布在博客网站上的内容单元，是露出的主要载体。

**组成部分**:
| 元素 | 说明 |
|------|------|
| 标题 | 文章标题，包含关键词 |
| 摘要 | 文章简介，150字左右 |
| 正文 | 主体内容，HTML格式 |
| 特色图片 | 文章封面图 |
| 分类 | 文章所属分类 |
| 发布时间 | 站内显示的发布日期 |
| 产品模块 | 推荐产品展示区（可选） |

---

### 产品模块（Product Module）

**定义**: 文章中展示推荐产品的区块，包含产品图片、名称、描述、价格和购买链接。

**数据结构**:
```json
{
  "name": "产品名称",
  "image": "产品图片URL",
  "description": "产品描述",
  "price": "$29.99",
  "link": "追踪链接"
}
```

**注意**: 所有9个网站现已支持产品模块。

---

### 特色图片（Featured Image / Hero Image）

**定义**: 文章的主图/封面图，用于列表页展示和文章详情页顶部。

**来源**: 从商家URL自动获取，或使用Unsplash等图库

**规格建议**: 1200×600px，16:9或2:1比例

---

### 文章图片体系（Article Images）

**定义**: 每篇文章固定包含的5张图片组合。

**图片构成**:

| 编号 | 类型 | 用途 | 位置 |
|------|------|------|------|
| `heroImage` | 头图 | 文章封面/列表页缩略图 | 文章顶部 |
| `image1` | 内容图1 | 正文配图 | 第1-2段后 |
| `image2` | 内容图2 | 正文配图 | 第3-4段后 |
| `image3` | 内容图3 | 正文配图 | 第5-6段后 |
| `image4` | 内容图4 | 正文配图 | 结尾前 |

**数据结构**:
```json
{
  "id": 1,
  "title": "文章标题",
  "images": {
    "hero": {
      "url": "https://example.com/hero.jpg",
      "local": "images/articles/1/hero.jpg",
      "alt": "文章主图描述"
    },
    "content": [
      {
        "url": "https://example.com/img1.jpg",
        "local": "images/articles/1/1.jpg",
        "alt": "内容图1描述"
      },
      {
        "url": "https://example.com/img2.jpg",
        "local": "images/articles/1/2.jpg",
        "alt": "内容图2描述"
      },
      {
        "url": "https://example.com/img3.jpg",
        "local": "images/articles/1/3.jpg",
        "alt": "内容图3描述"
      },
      {
        "url": "https://example.com/img4.jpg",
        "local": "images/articles/1/4.jpg",
        "alt": "内容图4描述"
      }
    ]
  },
  "content": "..."
}
```

**字段说明**:
- `url`: 原始外部链接（商家图片URL）
- `local`: 本地存储路径（下载到GitHub后的路径）
- `alt`: 图片描述（SEO用）

**存储策略（A+B混合方案）**:
- **默认**: 使用 `url` 外部链接（零存储成本）
- **可选**: 下载图片到 `local` 路径（永久保存）
- **检测**: 每天早上4:00自动检测外链有效性

**外链检测机制**:
| 项目 | 说明 |
|------|------|
| 检测时间 | 每天 04:00 (UTC+8) |
| 检测方式 | HEAD请求检测HTTP状态码 |
| 有效判定 | 状态码200且Content-Type为图片 |
| 无效处理 | 标记为失效，发送告警通知 |
| 告警方式 | 平台内通知 + 邮件（可选） |

**图片状态字段**:
```json
{
  "images": {
    "hero": {
      "url": "https://example.com/hero.jpg",
      "local": null,
      "alt": "主图描述",
      "status": "valid",
      "lastCheck": "2026-02-13T04:00:00Z"
    }
  }
}
```

**状态值**:
| 状态 | 说明 | 显示 |
|------|------|------|
| `valid` | 外链有效 | 🟢 正常 |
| `invalid` | 外链失效 | 🔴 需更换 |
| `local` | 已下载本地 | 🔵 本地存储 |
| `unchecked` | 未检测 | ⚪ 待检测 |

**本地存储目录结构**:
```
网站仓库/
├── images/
│   └── articles/
│       ├── 1/
│       │   ├── hero.jpg
│       │   ├── 1.jpg
│       │   ├── 2.jpg
│       │   ├── 3.jpg
│       │   └── 4.jpg
│       ├── 2/
│       │   └── ...
│       └── ...
└── js/
    └── articles/
        └── 1.json
```

### 文章分类（Category）

**定义**: 文章所属的内容类别，用于网站导航和内容组织。

**常见分类**:
| 英文 | 中文 |
|------|------|
| fashion | 时尚配饰 |
| beauty / health | 健康美容 |
| home | 家居园艺 |
| travel | 旅行住宿 |
| finance | 金融保险 |
| food | 美食饮品 |

---

### 发布时间（Publish Date）

**定义**: 文章在网站上显示的发布日期。

**默认规则**: 创建日期 **- 7天**

**示例**: 2026.2.12创建 → 站内显示 2026.2.5

**原因**: 让文章看起来不是刚发布的，增加可信度

---

### SEO（Search Engine Optimization）

**定义**: 搜索引擎优化，使内容更容易被搜索引擎收录和排名。

**露出相关SEO要点**:
- 标题包含关键词
- 摘要包含关键词
- 正文关键词密度适中
- 图片有alt属性
- URL友好（slug）

## 四、平台功能术语

### 内容创建（Content Creation）

**定义**: 在露出平台中创建新的博客文章的过程。

**流程**:
1. 输入商家URL
2. 输入追踪链接
3. 选择目标网站
4. 设置参数（关键词次数、发布时间）
5. AI生成内容
6. 预览和编辑
7. 保存/提交审核

---

### AI生成（AI Generation）

**定义**: 利用Claude API根据商家信息和露出提示词自动生成文章内容。

**生成内容包括**:
- 文章标题
- 文章摘要
- 正文内容（HTML）
- 产品推荐（如有）
- SEO元数据

---

### 审核（Review）

**定义**: 在内容发布前对其进行检查和确认的过程。

**审核人员**: wj02、wj07

**审核操作**:
| 操作 | 说明 |
|------|------|
| 通过 | 内容合格，可以发布 |
| 驳回 | 内容需修改，附带修改意见 |

**自检模式**: 审核人员自己创建的内容可跳过审核，仅需自检确认。

---

### 发布（Publish）

**定义**: 将审核通过的内容推送到目标博客网站的过程。

**技术实现**:
1. 生成JSON文件
2. 通过GitHub API推送
3. Cloudflare自动部署
4. 记录发布日志

---

### 草稿（Draft）

**定义**: 已创建但尚未提交审核的内容状态。

**特点**: 仅创建者可见，可随时编辑

---

### 待审核（Pending Review）

**定义**: 已提交等待审核人员处理的内容状态。

---

### 已通过（Approved）

**定义**: 审核通过，等待发布的内容状态。

---

### 已驳回（Rejected）

**定义**: 审核未通过，需要修改的内容状态。

---

### 已发布（Published）

**定义**: 已成功推送到目标网站的内容状态。

---

## 五、工作流程术语

### 露出工作流（Exposure Workflow）

**定义**: 从接收任务到内容发布的完整工作流程。

```
接收任务 → 创建内容 → 提交审核 → 审核处理 → 发布上线
    │           │           │           │           │
    │           │           │           │           └─ 记录日志
    │           │           │           │
    │           │           │           ├─ 通过 → 待发布
    │           │           │           └─ 驳回 → 修改 → 重新提交
    │           │           │
    │           │           └─ 自检模式（审核员）→ 直接发布
    │           │
    │           └─ AI生成 + 人工编辑
    │
    └─ 获取商家URL + 追踪链接
```

---

### 迁移（Migration）

**定义**: 将博客网站的数据存储格式从单一大文件改为每篇文章独立文件的过程。

**迁移前**: `js/data.js`（所有文章在一个文件）

**迁移后**:
- `js/articles-index.js`（索引文件）
- `js/articles/*.json`（单篇文章文件）

**目的**: 支持大规模文章存储、避免Git冲突、方便露出平台管理

---

### 推送（Push）

**定义**: 通过GitHub API将内容提交到代码仓库的操作。

**触发后果**: Cloudflare自动部署，网站更新

---

### 回滚（Rollback）

**定义**: 当发布出现问题时，将网站恢复到之前状态的操作。

**方法**: 使用备份文件恢复，或通过Git回退

---

## 六、数据结构术语

### 文章索引（Articles Index）

**定义**: 包含所有文章摘要信息的数据文件，用于列表页快速加载。

**文件**: `js/articles-index.js`

**包含字段**:
- id、title、slug
- category、categoryName
- date、image、excerpt
- featured、author、readTime
- hasProducts（是否有产品模块）

---

### 文章JSON（Article JSON）

**定义**: 单篇文章的完整数据文件，包含正文和产品信息。

**文件**: `js/articles/{id}.json`

**包含字段**:
- 索引字段（同上）
- content（正文HTML）
- products（产品数组）
- 其他扩展字段

---

### 统一数据格式（Unified Data Format）

**定义**: 团队为所有博客网站制定的标准化文章数据结构。

```json
{
  "id": 1,
  "title": "文章标题",
  "slug": "article-slug",
  "category": "fashion",
  "categoryName": "Fashion & Accessories",
  "date": "2026-02-05",
  "author": "Author Name",
  "excerpt": "文章摘要...",
  "image": "https://...",
  "content": "<p>HTML正文...</p>",
  "products": [
    {
      "name": "产品名",
      "image": "https://...",
      "description": "描述",
      "price": "$29.99",
      "link": "https://track..."
    }
  ],
  "featured": true,
  "readTime": "5 min read"
}
```
