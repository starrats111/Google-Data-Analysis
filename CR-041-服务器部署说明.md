## 🐛 故障排查

### 问题1：脚本执行失败

**症状：** `bash: deploy_cr041_complete.sh: command not found`

**解决方案：**
```bash
# 检查脚本是否存在
ls -la backend/scripts/deploy_cr041_complete.sh

# 赋予执行权限
chmod +x backend/scripts/deploy_cr041_complete.sh

# 重新执行
bash backend/scripts/deploy_cr041_complete.sh
```

### 问题2：服务启动失败

**症状：** `curl http://127.0.0.1:8000/health` 返回连接拒绝

**解决方案：**
```bash
# 查看详细日志
tail -100 /tmp/backend.log

# 检查端口是否被占用
lsof -i :8000

# 强制杀死占用端口的进程
fuser -k 8000/tcp

# 重新启动服务
cd ~/Google-Data-Analysis/backend
source venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &
sleep 3
curl -s http://127.0.0.1:8000/health
```

### 问题3：诊断脚本报错

**症状：** `python scripts/diagnose_l7d_data_accuracy.py` 报 Python 错误

**解决方案：**

```bash
# 检查数据库连接
python -c "from app.database import SessionLocal; db = SessionLocal(); print('✅ 数据库连接成功')"

# 检查模型导入
python -c "from app.models.user import User; print('✅ 模型导入成功')"

python scripts/diagnose_l7d_data_accuracy.py 2>&1
```

### 问题4：数据不准确

**症状：** 前端 L7D 数据与 Google Ads 不一致

**解决方案：**
```bash
# 查看 Google Ads 数据
python << 'EOF'
from app.database import SessionLocal
from app.models.google_ads_api_data import GoogleAdsApiData
from datetime import date, timedelta

db = SessionLocal()
end_date = date.today() - timedelta(days=1)
begin_date = end_date - timedelta(days=6)

data = db.query(GoogleAdsApiData).filter(
    GoogleAdsApiData.date >= begin_date,
    GoogleAdsApiData.date <= end_date
).all()

print(f'总记录数: {len(data)}')
for d in data[:10]:
    print(f'日期: {d.date}, 费用: {d.cost}, CPC: {d.cpc}, 预算: {d.budget}')
EOF
```
