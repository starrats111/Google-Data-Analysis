# 前端启动问题解决指南

## 问题：localhost:3000 无法连接

这是因为前端服务还没有启动。需要先安装Node.js和依赖，然后启动前端服务。

## 解决步骤

### 1. 安装 Node.js

如果系统提示 `npm 无法识别`，说明需要先安装 Node.js：

1. **下载 Node.js**
   - 访问：https://nodejs.org/
   - 下载 LTS 版本（推荐 18.x 或更高版本）
   - 安装时选择"Add to PATH"选项

2. **验证安装**
   打开新的命令行窗口，运行：
   ```bash
   node --version
   npm --version
   ```
   如果显示版本号，说明安装成功。

### 2. 安装前端依赖

在项目根目录下，运行：

```bash
cd frontend
npm install
```

这个过程可能需要几分钟，会下载所有必需的依赖包。

### 3. 启动前端服务

依赖安装完成后，运行：

```bash
npm run dev
```

如果成功，你会看到类似这样的输出：
```
  VITE v5.0.8  ready in 500 ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

### 4. 访问网站

打开浏览器，访问：http://localhost:3000

## 常见问题

### Q: npm install 很慢或失败？
A: 可以尝试使用国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q: 端口3000已被占用？
A: 可以修改 `frontend/vite.config.js` 中的端口号：
```javascript
server: {
  port: 3001,  // 改为其他端口
}
```

### Q: 启动后页面空白？
A: 检查：
1. 后端服务是否已启动（http://localhost:8000）
2. 浏览器控制台是否有错误信息
3. 网络请求是否正常

## 完整启动流程

### 后端（需要先启动）
```bash
cd backend
pip install -r requirements.txt
python scripts/init_db.py
python scripts/init_users.py
python scripts/init_platforms.py
uvicorn app.main:app --reload
```

### 前端（在另一个终端窗口）
```bash
cd frontend
npm install
npm run dev
```

## 注意事项

1. **需要两个终端窗口**：一个运行后端，一个运行前端
2. **先启动后端**：前端需要连接后端API
3. **保持服务运行**：不要关闭终端窗口，否则服务会停止

## 如果仍然无法连接

1. 检查防火墙设置
2. 检查是否有其他程序占用端口3000
3. 查看终端中的错误信息
4. 确认Node.js版本（需要16+）
